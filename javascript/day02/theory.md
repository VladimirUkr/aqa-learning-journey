# Day 02 — Итоги: методы массивов

## `.forEach()` — просто пройтись

```js
testCases.forEach((testCase) => {
    console.log(testCase);
});
```
Вызывает функцию для каждого элемента. Ничего не возвращает (в отличие от map/filter/reduce). Используй, когда нужно что-то *сделать* с каждым элементом, а не получить новый массив.

## `.map()` — преобразовать каждый элемент

```js
const doubled = numbers.map((number) => number * 2);
```
Возвращает **новый массив той же длины**, где каждый элемент — результат функции. Исходный массив не меняется.

## `.filter()` — отобрать часть элементов

```js
const errors = statusCodes.filter((code) => code >= 400);
```
Функция должна вернуть `true`/`false`. Возвращает новый массив (может быть короче исходного) только с элементами, где было `true`.

## `.find()` — найти один элемент

```js
const firstError = statusCodes.find((code) => code >= 400);
```
Возвращает **первый** подходящий элемент (не массив), или `undefined`, если ничего не найдено. Останавливается на первом совпадении.

## `.reduce()` — свернуть массив в одно значение

```js
const sum = numbers.reduce((accumulator, number) => {
    return accumulator + number;
}, 0); // 0 — начальное значение аккумулятора
```
Аккумулятор передаётся от итерации к итерации, накапливая результат. В конце возвращает финальное значение аккумулятора.

### Паттерн: reduce → объект-счётчик

Когда нужно посчитать количество вхождений каждого значения:
```js
const counts = results.reduce((accumulator, result) => {
    if (accumulator[result]) {
        accumulator[result] = accumulator[result] + 1;
    } else {
        accumulator[result] = 1;
    }
    return accumulator;
}, {}); // начальное значение — ПУСТОЙ ОБЪЕКТ
```
Ключевые моменты: начальный аккумулятор — `{}`, не `0`; `accumulator[key]` — bracket notation, обращение к свойству по значению переменной; обязательный `return accumulator` в конце каждой итерации, иначе следующая итерация получит `undefined`.

## Method chaining — соединение методов подряд

Каждый из `map`/`filter` возвращает новый массив, поэтому на результате сразу можно вызвать ещё один метод:
```js
const failedNames = testResults
    .filter((result) => result.status === "failed")
    .map((result) => result.name);
```
`forEach` в цепочку не годится — он возвращает `undefined`, продолжить цепочку на нём нельзя.

## Сравнение методов

| Метод | Возвращает | Назначение |
|---|---|---|
| `forEach` | `undefined` | просто пройтись, сделать что-то с каждым элементом |
| `map` | новый массив (той же длины) | преобразовать каждый элемент |
| `filter` | новый массив (может быть короче) | отобрать элементы по условию |
| `find` | один элемент (или `undefined`) | найти первое совпадение |
| `reduce` | одно значение (число/строка/объект/массив) | свернуть весь массив в результат |
