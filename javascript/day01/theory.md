# Day 01 — Итоги

## 1. Переменные и типы

- `let` — значение можно менять; `const` — нельзя переприсвоить.
- `typeof` всегда возвращает **строку** с названием типа: `"string"`, `"number"`, `"boolean"`, `"undefined"`, `"object"`, `"function"`.
- Ловушка: `typeof null` → `"object"` (историческая ошибка в JS, просто запомнить).
- Необъявленное значение (`let x;` без присвоения) имеет значение `undefined`, соответственно `typeof x` → `"undefined"`.
- Переменные всегда объявлять через `let`/`const` — без них создаётся неявная глобальная переменная (плохая практика, источник багов).

## 2. Операторы сравнения

- `===` — строгое равенство: сравнивает значение **и тип**. `100 === "100"` → `false`.
- `==` — сравнение с приведением типов (coercion). `100 == "100"` → `true`.
- В тестовом коде почти всегда используем `===` / `!==` — предсказуемее, меньше скрытых багов.

## 3. Логические операторы

- `&&` (AND) — истина, только если **все** условия истинны. Используется для проверки нескольких предусловий одновременно (например: `isLoggedIn && hasBalance`).
- `||` (OR) — истина, если **хотя бы одно** условие истинно. Используется для fallback/альтернативной логики (например: `hasPromoCode || hasGiftCard`).

## 4. Условия `if / else if / else`

- Цепочка останавливается на **первом** истинном условии — остальные `else if` не проверяются.
- Важно: если есть "совмещённое" условие (например, "делится и на 3, и на 5"), его нужно проверять **первым**, иначе более общее условие (просто "делится на 3") перехватит его раньше.

```js
if (i % 3 === 0 && i % 5 === 0) {
    console.log("FizzBuzz");
} else if (i % 3 === 0) {
    console.log("Fizz");
} else if (i % 5 === 0) {
    console.log("Buzz");
} else {
    console.log(i);
}
```

## 5. Функции

- Функция позволяет не копировать один и тот же код много раз — при изменении логики правится одно место, а не каждое использование.

## 6. Циклы

### `for` — когда нужен индекс или известное число итераций
```js
for (let i = 0; i < array.length; i++) {
    console.log(i, array[i]);
}
```
- Три части: инициализация; условие (проверяется перед каждым проходом); шаг.
- Частая ошибка (off-by-one): `i <= array.length` вместо `i < array.length` — даёт `undefined` на последней итерации.

### `while` — когда число итераций заранее неизвестно
```js
let attempt = 1;
while (attempt <= 5) {
    console.log(`Attempt ${attempt}`);
    attempt++;
}
```
- Условие проверяется **до** каждого прохода.
- Классика для автотестов: retry-логика, поллинг, ожидание элемента.
- Если забыть увеличить счётчик — бесконечный цикл.

### `do...while` — тело выполняется минимум один раз
```js
let attempt = 1;
do {
    console.log(`Attempt ${attempt}`);
    attempt++;
} while (attempt <= 5);
```
Условие проверяется **после** первого прохода.

### `for...of` — перебор значений (массивы, строки, Map, Set)
```js
for (const code of statusCodes) {
    console.log(code); // сами значения
}
```

### `for...in` — перебор ключей объекта (НЕ для массивов)
```js
const user = { name: "Vlad", role: "QA" };
for (const key in user) {
    console.log(key, user[key]);
}
```
Ловушка на массивах: `for...in` перебирает все enumerable-свойства, включая посторонние, добавленные вручную (`arr.label = "..."`), и отдаёт индексы как строки. Для массивов почти всегда нужен `for...of`, не `for...in`.

### `break` / `continue`
- `continue` — пропустить текущую итерацию, перейти к следующей.
- `break` — прервать цикл полностью.

### Вложенные циклы
```js
for (const suite of testSuites) {
    for (const testCase of suite) {
        console.log(testCase);
    }
}
```
`break`/`continue` по умолчанию влияют только на ближайший (внутренний) цикл.

## 7. Массивы (основы)

- `.push(value)` — добавляет элемент в конец.
- `.pop()` — удаляет и возвращает последний элемент.
- `array[array.length - 1]` — последний элемент по индексу.
- `.includes(value)` — проверка наличия значения.

## 8. Практика дня — решённые задачи

**FizzBuzz** (1-20): комбинация `for` + `if/else if/else` + `%` + `&&`.

**Retry с ранним выходом** (`while` + `break`):
```js
let attempt = 1;
let elementFound = false;

while (attempt <= 5) {
    console.log("Attempt " + attempt + ": not found");
    if (attempt === 4) {
        elementFound = true;
        console.log("Element found on attempt " + attempt + "!");
        break;
    }
    attempt++;
}
```

**Валидация паролей со счётчиком** (`for` + `if/else if/else` + счётчик):
```js
const passwords = ["abc", "Password1!", "12", "qwerty123", "P@ss"];
let okCount = 0;

for (let i = 0; i < passwords.length; i++) {
    const password = passwords[i];
    if (password.length < 6) {
        console.log(`${password}: is too short`);
    } else if (password.length >= 10) {
        console.log(`${password}: is too long`);
    } else {
        console.log(`${password}: OK`);
        okCount++;
    }
}

console.log(`Total OK passwords: ${okCount}`);
```
Счётчик объявляется **до** цикла (чтобы не сбрасывался на каждой итерации) и увеличивается только в нужной ветке — тот же паттерн, что используется в отчётах тестов ("сколько тестов прошло / упало").

## 9. Частые ошибки — шпаргалка

| Ошибка | Почему это баг |
|---|---|
| Сравнение строки с числом напрямую вместо `.length` | `"1!" === 8` всегда `false` |
| `typeof x` ожидается как `true`/`false`/`null` | На самом деле всегда строка-название типа |
| `for...in` на массиве | Может подхватить посторонние свойства, индексы — строки |
| Переменная без `let`/`const` | Создаёт неявную глобальную переменную |
| `while` без обновления счётчика | Бесконечный цикл |
| Проверка общего условия раньше совмещённого в `if/else if` | Совмещённое условие никогда не сработает |
